# 障害記録票

- id: 0006
- 修正日時: 2026/04/02 22:15:01

## 事象概要

- サーバー再起動後にトップページでログイン導線が表示されない（または表示が崩れる）事象が発生する可能性があった。

## 発生事象

- 対象ファイル:
  - `auth.ts`
- `getAppSession()` が `getServerSession(authOptions)` をそのまま返す実装だったため、セッション取得時に例外が発生したケースで呼び出し元ページ（`app/page.tsx` など）が例外伝播により正常描画できなくなる可能性があった。
- 例外発生時に未ログイン扱いへフォールバックする処理が存在しなかった。

## 直接原因

- セッション取得失敗時の例外ハンドリング不足。

## 根本原因

- 認証基盤呼び出しが常に成功する前提でページ描画フローを設計しており、再起動直後や環境変数不整合時などの失敗系を設計・テストで十分にカバーできていなかった。

## 対応方法

- 恒久対応:
  - `auth.ts` の `getAppSession()` で `getServerSession` 呼び出しを `try/catch` し、失敗時は `null` を返して未ログイン扱いへフォールバックするよう修正。
  - `test/unit/auth.test.ts` に以下のテストを追加。
    - `getServerSession` 成功時にセッションを返すこと。
    - `getServerSession` 失敗時に `null` を返すこと。

## エスカレーション

- 認証・セッション取得処理は「失敗時フォールバック」を標準パターンとして実装レビュー項目に追加する。
- ページ表示に影響する認証ユーティリティは、成功系だけでなく失敗系（例外）テストを必須化する。
